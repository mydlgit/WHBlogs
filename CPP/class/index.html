拷贝构造函数与拷贝赋值

class Test{
public:
    Test(){};
    Test& operator=(const Test&){ std::cout << "执行了赋值操作" << std::endl; return *this; };
    Test(const Test&){ std::cout << "执行了拷贝操作" << std::endl; }
    ~Test(){};
};
int main(){
//初始化
    Test t1;
    Test t2(t1);
    Test t3 = t1;
    Test t4, t5;
// 赋值
    t5 = t4 = t1;
    return 0;
}

 Test t2(t1); 属于直接初始化：要求编译器使用普通的函数匹配来选择与我们提供的参数最匹配的构造函数,因此直接调用拷贝构造函数来初始化对象也属于直接初始化。
 Test t3 = t1; 属于拷贝初始化，拷贝初始化首先使用指定构造函数创建一个临时对象，然后用拷贝构造函数将那个临时对象复制到正在创建的对象。（所以当复制构造函数被声明为私有时，所有的复制初始化都不能使用。） 这里t1已经存在，因此省略了构造临时对象的一步。
 使用=拷贝初始化时，要求编绎器将右侧运算对象拷贝到正在创建的对象中，可能需要进行类型转换，因此拷贝构造函数不应该是explicit。
 拷贝构造函数什么时候会被调用：

  • 用 = 定义变量时;
  • 将一个对象作为实参传递给一个非引用类型的形参;
  • 从一个返回类型为非引用类型的函数返回一个对象;
  • 用花括号列表初始化一个数组中的元素或一个聚合类中的成员;
  • 某些类类型还会对它们所分配的对象使用拷贝初始化。例如，当我们初始化标准库容器或是调用insert或push成员，容器会对其元素进行拷贝初始化。与之相反，用emplace成员创建的元素都进行直接初始化。

 赋值操作符返回一个指向操作符左侧对象的引用，这样可以实现链式操作，“t5=t4=t1”。

 拷贝构造函数的参数必须是引用类型，不一定是const类型，但在使用容器添加自定义类型时，如果自定义类没有实现移动构造函数，则要求拷贝构造函数的参数必须是const引用类型. 

class CExample  
{  
    int m_nTest;  
public:   
    CExample(int x):m_nTest(x) //带参数构造函数   
    {   
       cout << "constructor with argument/n";  
    }  
    CExample(const CExample & ex) //拷贝构造函数   
    {  
        m_nTest = ex.m_nTest;  
        cout << "copy constructor/n";  
    }  
    CExample& operator = (const CExample &ex)//赋值函数(赋值运算符重载)   
    {     
        cout << "assignment operator/n";  
        m_nTest = ex.m_nTest;  
        return *this;  
    }  
    void myTestFunc(CExample ex) {}
};

int main()  
{  
    CExample aaa(2);  // constructor with argument
    CExample bbb(3);  // constructor with argument
    bbb = aaa;          // assignment operator   
    CExample ccc = aaa;  // copy constructor  
    bbb.myTestFunc(aaa);  // copy constructor 
    return 0;     
}

 构造ccc，实质上是ccc.CExample(aaa); 我们假如拷贝构造函数参数不是引用类型的话， 那么将使得 ccc.CExample(aaa)变成aaa传值给ccc.CExample(CExample ex)，即CExample ex = aaa，因为 ex 没有被初始化， 所以 CExample ex = aaa 继续调用拷贝构造函数，接下来的是构造ex，也就是 ex.CExample(aaa)，必然又会有aaa传给CExample(CExample ex), 即 CExample ex = aaa;那么又会触发拷贝构造函数，就这下永远的递归下去。

所以， 为了避免拷贝构造函数的无限递归，拷贝构造函数是必须要带引用类型的参数的， 而且这也是编译器强制性要求的。
 需要析构函数 =>> 需要自定义拷贝构造函数和拷贝复制运算符; 需要拷贝构造函数<<=>>需要赋值操作

移动构造函数与移动赋值

 所谓移动语义，指的就是以移动而非深拷贝的方式初始化含有指针成员的类对象。简单的理解，移动语义指的就是将其他对象（通常是临时对象）拥有的内存资源“移为已用”, 提高初始化效率。对于一个包含指针成员的对象，移动语义即是对于该指针成员的浅拷贝，并且让源对象中的指针指向nullptr(因为此时新对象的指针和源对象的指针成员相同，如果源对象调用析构函数释放资源，会导致新对象的指针成员失去意义)。
 当类中同时包含拷贝构造函数和移动构造函数时，如果使用临时对象初始化当前类的对象，编译器会优先调用移动构造函数来完成此操作。只有当类中没有合适的移动构造函数时，编译器才会退而求其次，调用拷贝构造函数。 在实际开发中，通常在类中自定义移动构造函数的同时，会再为其自定义一个适当的拷贝构造函数，由此当用户利用右值初始化类对象时，会调用移动构造函数；使用左值（非右值）初始化类对象时，会调用拷贝构造函数。

   // move constructor
    MyString(MyString&& rhs)noexcept :_len(rhs._len), _data(rhs._data){
        ++MCtor;
        rhs._data=nullptr;
        rhs._len=0;
    }
    // move assignment
    MyString& operator = (MyString&& rhs) noexcept{
        ++MAsign;
        if(this!=&rhs){
            if(_data)delete[] _data;
            _data=rhs._data;
            _len = rhs._len;

            rhs._data=nullptr;
            rhs._len=0;
        }
        return *this;
    }

noexcept:
 移动构造函数在对象进行复制时通过直接移动原对象已经分配好的资源，可省去分配内存再拷贝的过程。在STL中大多数容器类型都会在调整容器大小（resize）时调用容器元素的移动构造函数来移动资源，但STL为了保证容器类型的内存安全，在大多数情况下只会调用被标记为不会抛出异常的移动构造函数，否则会调用其拷贝构造函数来作为替代。这是因为在资源的移动过程中如果抛出了异常，那么那些正在被处理的原始对象数据可能因为异常而丢失，而在拷贝构造函数中，由于进行的是资源拷贝操作，原始数据不会被更改，因此即使抛出异常也不会影响数据最终的正确性。

struct与class区别

 在c++中使用struct和class定义类的区别在于默认的访问权限：类可以在第一个访问说明符之前定义成员，若使用struct关键字，则这些成员属于public；若使用class，则属于private。
 此外class关键字定义的派生类默认是私有继承，而struct定义的则默认是公有继承。

隐式类类型转换(转换构造函数)

 如果类的构造函数只有一个实参，则它实际上定义了转换为此类类型的隐式转换机制；而关键字explicit则可以抑制这种隐式转换，且其只对一个实参的构造函数有效；用explicit声明构造函数时，它将只能以直接初始化的形式使用。  但是由于initializer_list的引入，对于multi-arg的构造函数，也可以发生上述的隐式类类型转换；此时如果用explicit声明构造函数，则要求只能通过直接初始化才构造对象。 

class C{
public:
    explicit C (int a, int b){cout << "C(int), a="<<a<<" ,b="<<b<<endl;} 
    C (const C& c){cout<<" copy ctor " << endl; }
};
int main(int argc, char const *argv[])
{
    C c{1, 2};
    C c2 = {1, 2}; // error! copy-list-initialization cannot use a constructor marked 'explicit
    return 0;
}

OOP

Object Oriented Programming

  • 派生类在声明时无需包含类派生列表；
  • 如果我们想将某个类用作基类，则该类必须已经定义而非仅仅声明。
  • final:
    在类名后加上finla关键字，表明该类不可被继承；若把函数指定为final，则该函数不能被派生类覆盖。
  • 静态类型：表达式的静态类型在编译时总是已知的，它是变量声明时的类型或者表达式生成的类型；
    动态类型则是变量或表达式表示的内存的对象的类型，动态类型直到运行时才可知。
    若表达式既不是引用也不是指针，则它的动态类型永远与静态类型一致。
  • 从派生类向基类的类型转换只对指针或者引用类型有效；
    不存在从基类向派生类的隐式类型转换，因为一个基类的对象可能是派生类对象的一部分，也可能不是，所以不存在从基类向派生类的自动类型转换。
  • 如果用一个派生类对象为一个基类对象初始化或赋值时，只有该派生类对象中过的基类部分会被拷贝、移动、赋值，它的派生类部分会被忽略。
  • class derive: public base{...}
    其中public称为派生访问说明符，其控制派生类用户（即派生类对象和派生类的派生类对象）对于基类成员的访问权限； 此外还可以控制继承自派生类的新类的对于基类成员的访问权限。（见inherit/继承关系）
  • 改变个别成员的可访问性：在派生类内部使用using说明，可以将其直接或间接基类中的任何可访问成员标记出来，using声明语句中名字的访问权限由该using声明语句前的访问说明符决定，与派生访问说明符无关。
    class derived: private base{
    public:
    using base::var1;
    private:
    using base::var2;
}

 虽然derived使用了私有继承，但var1 仍然可以被derived的所有用户访问；而var2则只能被derived的成员和友元访问。

  • 一个对象、引用或者指针的静态类型决定了该对象的哪些成员是可见的。
    即使静态类型与动态类型可能不一致(例如使用基类的引用或指针)，但是我们能使用哪些成员仍然是由静态类型决定的。
    定义派生类中的函数不会重载其基类中的成员，和其他作用于域一样，如果派生类的成员与基类的某个成员同名，则派生类将在其作用域内隐藏该基类成员。即使派生类成员函数和基类成员函数的形参列表不一致，基类成员也仍然会被隐藏。
  • 定义派生类的拷贝或移动构造函数：需要使用对应的基类构造函数初始化对象的基类部分，若不提供初始值，则基类部分是被默认初始化，而非拷贝或移动。
    // correct usage:
    class Base{...};
    class derived{
    derived(const derived& d): Base(d){}
    }
  • =delete:
    在函数参数列表后加上=delete来指出我们希望将它定义为删除的，其将不能以任何方式使用，可以用来阻止拷贝、赋值（如iostream类）。 如果一个类有数据成员不能默认构造、拷贝、复制或销毁，则对应的成员函数将定义为删除的。
  • 赋值运算符(operator=)：将一个对象赋予它自身时，必须能正常工作，因此一定要在operator=重载实现中检测是否是自我赋值(self assignment)；
    大多数赋值运算符组合了析构函数和拷贝构造函数的工作：即销毁左侧运算对象的资源和从右侧运算对象拷贝数据。
  • swap:对于自定义类上定义的swap，其优先级高于std中定义的版本。
  • 合成移动构造函数：只有当一个类没有定义任何自己版本的拷贝控制成员，且它的所有数据成员都能移动构造或移动赋值时，编译器才会为它合成移动构造函数或移动赋值运算符。
  • 如果一个类没有移动构造函数，通过正常的函数匹配，类会使用对应拷贝操作来代替移动，即使我们试图调用move来移动它们时也是如此。
  • 区分移动和拷贝的重载函数：通常拷贝构造函数接收一个const T&, 而移动拷贝构造函数接收一个T&&。

补充

  • 枚举类型 
    enum class open_modes {x, xx};  //限定作用域的枚举类型， 枚举成员的名字在枚举类型作用域外不可见
    enum color {red, yellow}; //不限定作用域的枚举类型，枚举成员作用域与枚举类型本身的作用域相同。
    //未命名enum: 只能在定义该enum时定义它的对象。
    enum {xxx} obj;

  • 嵌套类:定义在其他类内部的类，与外层类的对象是相互独立的；嵌套类的名字在外层类作用域中是可见的，在外层作用域之外不可见；若在嵌套类中声明静态成员，则该成员的定义将位于外层类的作用域之外。
    Type outer_class::inner_class::static_memb = xx;

  • 局部类:定义在某个函数的内部，只在定义它的作用域可见，且不允许声明静态数据成员。局部类只能访问外层作用域定义的类型名、静态变量、枚举成员，不能访问函数的普通局部变量。同时外层函数不能访问局部类的private member； 局部类的嵌套类也属于局部类。

  • 聚合类：所有成员都是public; 没有定义任何构造函数；没有类内初始值；没有基类，也没有virtual函数。
  • union:一种特殊的类，可以有多个数据成员，但是在任意时刻只有一个数据成员可以有值。当对某个成员赋值后，其他成员就变成未定义的状态。
  • 如果在一个构造函数中调用另一个构造函数 
    class A
    {
    public:
            A()
            {
                    printf("In A::(). m_x=%d\n", m_x);
                    A(0);   // 这里只是相当于声明了一个临时变量，不会改变m_x
                    // new(this) A(0), 使用placement new 来构建一个新对象，这里会改变m_x的值
                    printf("Out A::(). m_x=%d\n", m_x);

            }

            A(int x)
            {
                    printf("In A::(int x). x=%d\n", x);
                    m_x=x;
            }

    private:
            int m_x;
    };
