About iterator

迭代器类型

  • struct input_iterator_tag{}; 只能向前移动，如istream_iterator, 支持++iter或iter++, 对于所指对象只读;
struct output_iterator_tag{}; 只能向前移动，如ostream_iterator, 支持++iter或iter++, 对于所指对象只写;
struct forward_iterator_tag:: public input_iterator_tag{}; 只能向前移动，但对于所指对象同时可读可写
struct bidirectional_iterator_tag:: public forward_iterator{}; 可前后移动，
如map、set、multimap、multiset的迭代器。
    
struct random_access_iterator_tag:: public bidirectional_iterator_tag{}; 
随机访问迭代器，可进行迭代器算术运算, 容器元素存储在连续内存空间内, 如vector、string、deque。
//迭代器指示的新位置与原来相比向前（向后）移动n个元素，结果迭代器或指示容器中的一个元素或为尾后迭代器。
iter+n or iter-n  
iter+=n 或者 iter-=n;
    
//两个迭代器相减是它们之间的距离。参与运算的两个迭代器必须指向同一个容器中的元素或为尾后迭代器，
//其类型为difference_type的带符号整型数。
iter1-iter2 

  • iter:解引用迭代器，返回其所指元素的引用。 iter++: iter向后移动一位，返回一个iter移动前的临时对象，一个右值
    ++iter: 返回 iter移动后的引用, 一个左值，++iter=&variable;//正确，但其修改的是iter指针值，而非其指向的对象，容器内元素没有改变。
    iter++: 等价于 (iter++), 是一个左值
    ++iter: 等价于 *(++iter)

  • iterator：但凡使用了迭代器的循环体，都不要向迭代器所属的容器添加或者减少元素，即不要改变其对象容量大小。 
    /*
        实质是一个 int* const指针，表示iter不能指向其他对象，++iter会报错，但依旧所指向对象内容依旧可被修改
    */
    const vector<int>::iterator iter=vec.begin();

    /*
        实质是一个const int*指针
    */
    vector<int>::const_iterator citer = vec.cbegin()
