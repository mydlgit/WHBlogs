About new and delete

new

  先分配内存，再调用构造函数 

string* ps = new string("hello");
// 编译器会转化为以下形式
void* mem = operator new(sizeof(string));   // 分配内存
ps = static_cast<string*>(mem);     //转型
ps->string::string("hello");        //调用构造函数

delete

  先调用析构函数，再释放内存 

delete ps;
// 编译器会转化为以下形式
string::~string(ps);        // 析构函数
operator delete(ps);        // 释放内存

  array new 必须要搭配array delete(即delete []); 对于内置类型，如int、double, 采用delete也可，因为int、double类型没有析构函数，内存会被正常释放；对于类类型，如果该类没有提供析构函数，且该类的成员中不包含类类型，只有内置类型，采用delte也不会有内存泄漏；但如果其中包含类类型成员或指针成员，则delete只会释放数组中第一个元素占用的内存，其余元素内存将会泄漏，此时程序不会报错；
 或若如下面情况，Demo中提供了析构函数(即使函数体为空)，无论类成员类型是内置类型，还是类类型、指针类型，由于内存布局中包含了数组长度，采用delete来试图释放内存都会报错。

  • 重复delete
     delete一次的指针并非NULL，而是成了空悬指针(dangling pointer)，虽然指针已经无效，但在很多机器上指针仍然保存着(已经释放了的)动态内存的地址。
      重复delete会造成不可知的错误，虽然编译器可能会检测出重复delete的发生，但不要依赖编译器自身的选择。
      对于delete一次的指针，可以显示令其为NULL，清楚指出该指针不指向任何对象，delete一个空指针不会有任何问题
    int\* p = new int(1);
    delete p;
    p=NULL;
    delete p; //删除一个空指针，不会报错

关于重载

    class FF
    {
    public:
        FF() :_id(0) { std::cout << "default ctor.this=" << this << " id=" << _id << std::endl; }
        FF(int i) :_id(i) { std::cout << "ctor.this=" << this << " id=" << _id << std::endl; }
        ~FF() { std::cout << "dtor.this=" << this << " id=" << _id << std::endl; }
        static void* operator new(size_t size);
        static void operator delete(void* pdead, size_t size);
        static void* operator new[](size_t size);
        static void operator delete[](void* pdead, size_t size);
    private:
        int _id;
    };

    void* FF::operator new(size_t size)
    {
        FF* p = (FF*)malloc(size);
        std::cout << "调用了Foo::operator new" << std::endl;
        return p;
    }

    void FF::operator delete(void* pdead, size_t size)
    {
        std::cout << "调用了Foo::operator delete" << std::endl;
        free(pdead);
    }
    void* FF::operator new[](size_t size)
    {
        FF* p = (FF*)malloc(size);
        std::cout << "调用了Foo::operator new[]" << std::endl;
        return p;
    }

    void FF::operator delete[](void* pdead, size_t size)
    {
        std::cout << "调用了Foo::operator delete[]" << std::endl;
        free(pdead);
    }

    // main
    FF* pf = new FF(7);
    FF* pf1 = new FF[3];
    delete pf;
    delete[] pf1;
    // 输出
    // 调用了Foo::operator new
    // ctor.this=011D62B8 id=7
    // 调用了Foo::operator new[]
    // default ctor.this=011E0DCC id
    // default ctor.this=011E0DD0 id
    // default ctor.this=011E0DD4 id
    // dtor.this=011D62B8 id=7
    // 调用了Foo::operator delete
    // dtor.this=011E0DD4 id=0
    // dtor.this=011E0DD0 id=0
    // dtor.this=011E0DCC id=0
    // 调用了Foo::operator delete[]

 通过::new和::delete可以强制使用全局的operator new和operator delete，而非重载版本。
 类中的operator new和operatir delete的重载函数必须是static，因为operator new就是在创建对象时被调用，此时尚未有对象存在，如果函数是非静态的，自然是不能被调用的。

总结

new/delete and operator new/delete and placement new
new/delete属于操作符，而operator new/operator delete是函数.

  • new:
    调用operator new分配足够的空间，并调用相关对象的构造函数;
    不可以被重载
  • operator new 
    （1）只分配所要求的空间，不调用相关对象的构造函数。当无法满足所要求分配的空间时，则
        ->如果有new_handler，则调用new_handler，否则
        ->如果没要求不抛出异常（以nothrow参数表达），则执行bad_alloc异常，否则
        ->返回0
    （2）可以被重载
    （3）重载时，返回类型必须声明为void*
    （4）重载时，第一个参数类型必须为表达要求分配空间的大小（字节），类型为size_t
    （5）重载时，可以带其它参数，即所谓的placement new，
    形如operator new(size_t, string)、operator new(size_t, void*)形式;  
     (6)placement new对应形式的重载operator delete也可叫做placement delete, 但其正常情况下不会被调用，
    只有当相应版本的placement new调用构造函数过程中出现异常时，才会被调用(事实上不同编译器对此有不同表现)，
    用来归还未完全创建成功的object所占用的memory；
    若缺少对应版本的placement delete,并不会报错，意味着放弃构造函数抛出的异常。
    如果placement new构造成功后, 调用delete释放内存, 
    所调用的是正常形式的operator delete(可能是全局的，也可以是自定义的)。
  • 接受第二个指针参数的placement new, 执行时忽略size_t参数，只返还第二个参数，一个指向已经分配好的某段内存的一个指针，这段内存可能在栈中或者静态全局区，而delete只能释放分配在堆中的内存，因此通过此种placement new得到的指针不能调用delete，只需调用指向对象的析构函数。 
    new (place_address) type
    new (place_address) type (initializers)
    new (place_address) type [size]
    new (place_address) type [size] { brace initializer list}

 其中place_address是一个指针， 同时在initializers提供的一个(可为空)以逗号分隔的初始值列表，该初始值列表将用于构造新分配的对象。 当只传入一个指针类型的实参时，定位new表达式构造对象但是不分配内存。这种重载形式的operator new常被翻译为定位new，C++库中已经提供了该版本。

优点: 

    1.用placement new 解决buffer的问题
    问题描述:用new分配的数组缓冲时,由于调用了默认构造函数,因此执行效率上不佳。若没有默认构造函数则会发生编译时错误。
    如果你想在预分配的内存上创建对象，用缺省的new操作符是行不通的。要解决这个问题，
    你可以用placement new构造。它允许你构造一个新对象到预分配的内存上。

    2.增大时空效率的问题
    使用new操作符分配内存需要在堆中查找足够大的剩余空间，显然这个操作速度是很慢的，
    而且有可能出现无法分配内存的异常（空间不够）。
    placement new就可以解决这个问题。我们构造对象都是在一个预先准备好了的内存缓冲区中进行，不需要查找内存，
    内存分配的时间是常数；而且不会出现在程序运行中途出现内存不足的异常。
    所以，placement new非常适合那些对时间要求比较高，长时间运行不希望被打断的应用程序。
