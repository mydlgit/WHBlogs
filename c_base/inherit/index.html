关于继承

1. protected

 基类中被声明为protected的成员，对于类的用户是不可访问的；但对于派生类的成员和友元是可访问的; 需注意的是，派生类的成员函数只能访问所作用的那个对象（即this指针指向的对象）的基类保护成员，不能访问其他基类对象的基类保护成员。在派生类的成员函数外部，不能访问基类的私有成员和保护成员。 

#include<iostream>
using namespace std;

class A{
public:
  int a;
  A(){
    a1 = 1;a2 = 2;a3 = 3;a = 4;
  }
public:
  int a1;
protected:
  int a2;
private:
  int a3;
};
class B : protected A{
public:
  int a;
  B(int i){
    A();
    a = i;
  }
  void fun(){
    cout << a << endl;        //正确，public成员，输出i。
    cout << a1 << endl;       //正确，基类的public成员，在派生类中变成了protected，可以被派生类访问。
    cout << a2 << endl;       //正确，基类的protected成员，在派生类中还是protected，可以被派生类访问。
    cout << a3 << endl;       //错误，基类的private成员不能被派生类访问。
  }
};
int main(){
  B b(10);
  cout << b.a << endl;       //正确。public成员
  cout << b.a1 << endl;      //错误，protected成员不能在类外访问。
  cout << b.a2 << endl;      //错误，protected成员不能在类外访问。
  cout << b.a3 << endl;      //错误，private成员不能在类外访问。
  return 0;
}

2. 继承关系

 派生类在继承基类时，可以指定public、protected、private三种继承方式：

  • public继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：public, protected, private
  • protected继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：protected, protected, private
  • private继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：private, private, private

 但无论哪种继承方式，基类中的private成员只能被本类成员（类内）和友元访问，不能被派生类访问；以及protected成员可以被派生类访问。
 继承关系改变的是派生类的派生类对于基类成员的访问权限，以private继承为例，基类的public成员对于派生类的派生类对象也是不可访问的。

  • 重点说明
     以public 形式继承，编译器会在必要时刻，将派生类对象转换成基类对象；而以private形式继承，则不会发生这种行为。
     private 继承意味着implemented-in-terms-of(根据某物实现出)，即派生类与基类并不存在is-a的关系。
     当你面对并不存在is-a关系的两个classes，其中一个类需要访问另一个类protected成员，或者需要重新定义其一或多个virtual函数，可以考虑private继承。

3.多重继承

 允许派生类从它的一个或多个基类中继承构造函数，此外如果从多个基类中继承了相同的构造函数（即形参列表相同），则派生类必须定义专属版本的构造函数，与基类的形参列表不同。
 基类或派生类的合成拷贝控制成语、合成构造函数、赋值运算符或析构函数，它们对本身的成员依次进行初始化、赋值或销毁操作，此外还负责使用直接基类中对应的操作对一个对象的直接基类部分进行初始化、赋值或销毁直至继承链顶端。
 尽管在派生列表中同一个基类只能出现在一次，但派生类可以多次继承同一个类。派生类可以通过其多个基类分别继承同一个间接基类，也可以直接继承某个基类，然后通过另一个基类再一次间接继承该类。 在默认情况下，派生类中含有继承链上每个类对应的子部分，如果某个类在派生过程中出现了多次，则派生类中将包含该类的多个子对象。
 虚继承：令某个类做出声明，承诺愿意共享它的基类。其中共享的基类子对象称为虚基类。在这种机制下，不论虚基类在继承体系中出现了多少次，在派生类中都只包含唯一一个共享的虚基类子对象。

  class Bear: virtual public ZooAnimal{}
  //virtual和public顺序随意， virtual说明了一种意愿，即在后续的派生类当中共享虚基类的同一份实例

虚派生只影响从指定了虚基类的派生进一步派生出的类，不会影响派生类本身。（eg.即影响的是Bear的派生类） 影响包括: 虚基类是由最底层的派生类初始化的；
虚基类成员的可见性:
假定B定义了一个名为x的成员，D1和D2都是从B虚继承得到的，D继承了D1和D2，则在D的作用域中，x通过D的俩个基类都是可见的，如果我们通过D的对象使用x，有三种可能性：
 1.如果在D1和D2中都没有x的定义，则x将被解析为B的成员，此时不存在二义性，一个D的对象只含有x的一个实例；
 2.如果x是B的成员，同时是D1和D2中某一个的成员，则同样没有二义性，派生类的x比共享虚基类B的x优先级更高；
 3.如果在D1和D2中都有x的定义，则直接访问x将产生二义性问题；与非虚的多重继承体系一样，解决这种二义性问题最好的方法是在派生类中为成员自定义新的实例。

含有虚基类的对象的构造顺序与一般的顺序区别：首先使用提供给最底层派生类构造函数的初始值初始化该对象的虚基类部分，接下来按照直接基类在派生列表中出现的顺序依次进行初始化；
