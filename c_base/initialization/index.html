About Initialization

声明

  • 声明一个变量只是将变量名标识符的有关信息告诉编译器，使编译器“认识”该标识符，但声明不一定引起内存的分配。

定义

  • 定义变量意味着给变量分配内存空间，用于存放对应类型的数据，变量名就是对相应的内存单元的命名。而在C++程序中，大多数情况下变量声明也就是变量定义，声明变量的同时也就完成了变量的定义，不过也有例外。如果想声明一个变量而非定义它，则可加上extern关键字，而且不用显示初始化变量。
  • 任何包含了显示初始化的声明即成为定义, 即extern语句如果包含初始值就不再是声明，而变成定义了。 
extern int a; //仅仅声明一个全局变量 ，没有定义，
int a; //声明并定义一个全局变量
extern int a = 0;//定义全局变量并初始化初值 
int a = 0; //定义全局变量并初始化初值
  • 变量的定义必须出现且只能出现在一个文件中，而其他用到该变量的文件必须对其进行说明，但绝不能重复定义。当你要引用一个全局变量时，你就要声明extern int a;这个时候extern不能省，否则就成定义了。例如如果想要在多个文件之间共享const对象时，必须在变量的定义之前添加extern。
header.h
extern const int a =1;

main.cpp
main(){
extern const int a;
cout << a;  //输出1
}
  • 不能在任何函数体内部(包括main函数)初始化一个由extern关键字标记的变量。

初始化

  • 初始化不是赋值，其含义是创建一个变量时赋予其一个初始值，而赋值的含义是把对象当前值擦除，而以一个新值来替代。 
int a=1;    //初始化
a=2;        //赋值
  • 列表初始化， 要求提供的初始值与要初始化的变量类型严格统一，在使用列表初始化时，若初始值存在丢失信息的风险(如用long、double的值去初始化int变量，有可能丢失数据或者溢出)，则编译器将报错。 
int a ={0};  int b{0};

  • 初始化与定义，定义发生时不一定会进行初始化：
    当变量为自定义类类型:
     调用自身的默认构造函数进行初始化。
    当变量为内置类型时:
     该变量定义在函数体外时，将被初始化为0。
     该变量定义在函数体内时，将不会进行初始化, 一个未被初始化的内置类型变量的值是未定义的。

  • 类成员变量初始化：初始化列表和赋值初始化

  • 初始化列表:
    格式: 
Base(int a1, int a2, A a): data1(a1), data2(a2), data3(a);

 必须用初始化列表的情况：
 1）如果我们有一个类成员，它本身是一个类或者是一个结构，而且这个成员它只有一个带参数的构造函数，没有默认构造函数。这时要对这个类成员进行初始化，就必须调用这个类成员的带参数的构造函数。
 2）当类成员中含有一个const对象时，或者是一个引用时，他们也必须要通过成员初始化列表进行初始化，因为这两种对象要在声明后马上初始化，而在构造函数中，做的是对他们的赋值，这样是不被允许的。
 3）子类初始化父类的私有成员，需要在(并且也只能在)参数初始化列表中显示调用父类的构造函数。
 4)成员变量的初始化顺序与声明的顺序相关，与在初始化列表中的顺序无关；初始化列表先于构造函数的函数体执行。

  • 类内初始值：提供一个类内初始值时，必须以符合=或者花括号表示，不能用()括号；
    eg: vector f{type_item}; //正确
    vector f(n, type_item); //意图初始化包含n个type_item的vector，错误原因是容易和函数声明冲突；该句在类外使用时是可以的。

  • static member data initialization
     静态成员变量不属于类的任何一个对象，所以它们并不是在创建类的对象时被定义的，这意味着它们不是由类的构造函数初始化的。即在类内的静态成员变量声明并非定义，只有在类外定义或者初始化后，才能访问该静态成员变量。 所以static 成员变量不随对象的创建而分配内存，也不随对象的销毁而释放内存。而普通成员变量在对象创建时分配内存，在对象销毁时释放内存。通常情况下，不在类内初始化静态成员，而是在类外对其初始化，
    eg: 类内声明为 static vector< int > elems; 类外初始化为 vector< int > class_name::elems; 这里可以赋初值，也可以不赋。
    不过也可以为静态成员变量提供类内const初始值，但要求静态成员必须是字面值常量类型的constexpr，初始值也必须是常量表达式。
    eg. static contestexpr int period = 20; static const int _buf_size =1024;
    即使一个static member data在类内被初始化后，也应在类的外部定义一下该成员。

  • 值初始化与默认初始化
     值初始化顾名思义，就是用数值初始化变量。
     默认初始化：如果定义变量时没有指定初值，则变量被默认初始化。其初始值和变量的类型以及变量定义的位置相关。默认初始化类对象和默认初始化内置类型变量有所不同。
     对于默认初始化内置类型变量来说：定义在函数体之外的变量是全局变量，一般存储在全局区，存储在全局区的变量一般会执行值初始化。此时，其初始值和变量的类型有关。对于int类型其初始值为0，对于char类型其默认初始值为''。
     定义在函数体内部的是局部变量，其存储在栈区中，如果没有指定初值，那么该局部变量将不会被初始化，也就是说这个局部变量的值是未定义的，是个随机值。此时，如果不给这个局部变量赋值，那么就不能使用该局部变量，否则就会出错，注意这种情况是没有被初始化，既没有使用默认初始化也没有使用值初始化，没有初始化的值是不能使用的.

Uniform Initialization

 Since C++ 11, for any initialization, you can use one common syntax, that is brace, "{}". 

int va[] {1, 2, 3};
vector<int> vec {1, 2, 3};
complex<double> c{4.0, 2.0};

  编译器看到"{}"会将其视为initializer_list, 其又关联一个array(intializer_list内部有一个指向array的指针，拷贝一个initializer_list属于浅拷贝，造成两个对象中的指针指向同一个array)。调用构造函数时:

  • 若存在一个版本有多个参数，通过{}初始化时，关联的array内的元素可被编译器分解逐一传递给函数； 通过()初始化时，提供和构造函数相同数量、类型的参数，也可进行初始化。
  • 若存在一个版本接受initializaer_list类型的参数，则通过{}初始化时，会优先使用该版本。
class P{
     public:
          P(int a, b){cout<<"P(int,int), a="<<a<<" , b="<<b;}
          P(initializer_list<int> initlist){
               cout <<"P{initializer_list<int>}, vlaues=";
               for(auto& v: initlist){
                    cout<<v<<" ";
               }
          }
}

//测试
P p(1, 2);     // 输出P(int, int), a=1, b=2
P p2{1, 2};     // 输出P{initializer_list<int>}, values=1, 2
P p3{1, 2, 3};    // 输出P{initializer_list<int>}, values=1, 2, 3
P p4={1, 2, 2};  // 输出P{initializer_list<int>}, values=1, 2, 2
