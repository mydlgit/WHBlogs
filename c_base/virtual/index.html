About Virtual Function

虚函数实现了C++的多态

 在c++中，一个基类的指针或者引用可以指向或者引用派生类的对象。对于基类中的虚函数，其含义是：当通过基类的指针或者引用调用该成员函数时，若派生类覆盖(override)了基类中的虚成员函数，将根据指针实际指向的对象类型确定调用的函数，而非指针的类型。
 上述行为也可以称为动态绑定（运行时绑定），即函数的运行版本由实参决定，在运行时选择函数的版本。 

class Base{
public:
virtual void test(){ cout << “基类方法”};
virtual ~Base(){};
}
class Derived:public Base{
public:
void test(){ cout << “派生类方法”};
}
  • 只需将基类中的成员函数声明为虚函数即可，派生类中重写的virtual函数自动成为虚函数；

  • 派生类若存在和父类虚函数同名但参数类型不同的函数时，会导致父类中的这个虚函数对于子类对象不可见。此时可通过Derived.Base::fcn()来调用被隐藏的函数

  • 基类中的析构函数必须为虚函数，否则会出现对象释放错误。如果基类的析构函数不是虚函数，则删除一个基类的指针的时候，C++不管这个指针指向一个基类对象还是一个派生类的对象，调用的都是基类的析构函数而不是派生类的。如果实际指向的派生类对象依赖于派生类的析构函数的代码来释放资源，这样做就会造成资源泄漏。如果基类的析构函数是虚函数，并且派生类提供自己的析构函数，则删除基类指针时，会调用派生类的析构函数。

  • 虚函数的使用将导致类对象占用更大的内存空间。编译器给每一个包括虚函数的对象添加了一个隐藏成员：指向虚函数表的指针vptr。
     虚函数表是编译器在编译时设置的静态数组，属于类而非某个具体的实例。虚函数表包含可由类的对象调用的每个虚函数的一个条目。此表中的每个条目是一个函数指针，指向该类可访问的函数。
     vptr在创建类实例时自动设置，以便指向该类的虚函数表(与this指针不同，this指针实际上是编译器用来解析自引用的函数参数，vptr是一个真正的指针)。当派生类重新定义虚函数时，则将该函数的地址添加到虚函数表中。无论一个类对象中定义了多少个虚函数，虚函数指针只有一个。相应地，每个对象在内存中的大小要比没有虚函数时大4个字节（32位主机, 64位机器则为8字节)。

  • 构造函数不可以声明为虚函数：尽管虚函数表vtable是在编译阶段就已经建立的，但指向虚函数表的指针vptr是在运行阶段实例化对象时才产生的。 如果构造函数是虚的，就需要通过虚指针指向的虚函数表来调用，可是此时对象还没有实例化，尚未分配内存，就没有虚指针，所以构造函数不能是虚函数。

  • 静态函数不可以声明为虚函数: static成员函数不属于任何类对象或类实例，且虚函数依靠vptr和vtable来处理。vptr是一个指针，在类的构造函数中创建生成，通过this指针来访问它，静态成员函数没有this指针，所以无法访问vptr。

  • 析构函数可以声明为虚函数。如果我们需要删除一个指向派生类的基类指针时，应该把析构函数声明为虚函数。 事实上，只要一个类有可能会被其它类所继承， 就应该声明虚析构函数(哪怕该析构函数不执行任何操作)。 

    class A {
    public:
        virtual void vfunc1();
        virtual void vfunc2();
        void func1();
        void func2();
    private:
        int m_data1, m_data2;
    };

    class B : public A {
    public:
        virtual void vfunc1();
        void func1();
    private:
        int m_data3;
    };

    class C: public B {
    public:
        virtual void vfunc2();
        void func2();
    private:
        int m_data1, m_data4;
    };

  • 通过虚函数表调用虚函数 
class CTest {
public:
    virtual void Test1(void){cout<< "CTest Test1\n";}
    virtual void Test2(void){cout<< "CTest Test2\n";}
};

typedef void(*fun)();
void testT(){
    CTest cc;
    typedef long long int64;    // 64位机上，指针大小是8个Byte, 指针+1偏移64位长度
    int64* Vptr=(int64*)(&cc);  //虚指针值代表的是虚函数表入口地址
    int64* pVtbl = (int64*)*Vptr;       //虚函数表是一个数组，数组元素是函数指针，pVtbl相当于数组元素首地址
    int64* pFirstVf = (int64*)(pVtbl[0]);       //pVtbl[0] 等价于 *(pVtbl)
    int64* pFirstVf2 = (int64*)*pVtbl;
    int64* pSecondVf = (int64*)(pVtbl[1]);      //pVtbl[1] 等价于 *(pVtbl+1)
    int64* pSecondVf2 = (int64*)*(pVtbl+1);

    fun pf;
    pf = (fun)pFirstVf;
    pf();   // CTest Test1
    pf = (fun)pFirstVf2;
    pf();   //CTest Test1
    pf = (fun)pSecondVf;
    pf(); //CTest Test2
    pf = (fun)pSecondVf2;
    pf();   //CTest Test2
}
  • 多重继承机制下的虚函数表
     子类对象会有多个虚函数指针，指向不同的虚函数表，虚函数指针顺序按照派生顺序；如果子类改写了父类的虚函数，那么就会用子类自己的虚函数覆盖相应的父类虚函数；如果子类有新的虚函数，那么就添加到第一个虚函数表的末尾。

虚函数的静态决议

 在下面两种情况下，虚函数不会出现预期行为:

  • 在基类的constructor和destructor内，因为当我们构造派生类对象时，会首先调用基类的constructor，若基类中存在虚函数，则调用的是一定是基类自身的，而非派生类的，因为此时派生类的data members尚未初始化；同理，在基类deconstructor中存在虚函数也适用此规则，因为派生类的deconstructor先被调用，其date members已经被释放。
  • 当我们使用基类的对象，而非指针或引用时；为了能在单一对象中展现多种型别，多态需要一层间接性，在C++中，惟有以基类的指针和引用才能支持面向对象编程概念。当我们为基类声明一个实际对象时，同时也就配置出足以容纳该对象的实际内存空间，如果传入一个派生类对象，那么就不一定有足够的内存放置派生类的各个data members, 那些属于派生类而非基类的成员将被切掉。

纯虚函数

在普通的纯虚函数后面加上“=0”， eg. virtual void test() = 0;
 何时使用纯虚函数:

  • 当想在基类中抽象出一个方法，且该基类只做能被继承（即为抽象基类），而不能被实例化；
  • 这个方法必须在派生类(derived class)中被实现； 如果满足以上两点，可以考虑将该方法申明为pure virtual function; 可以为纯虚函数提供定义，不过函数体必须定义在类的外部。
